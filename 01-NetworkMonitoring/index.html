Network Monitoring

Why do we need this?

  • Token from producion non root build
  • Awarness of data flow in the app being tested
  • Ability to finding issues related to API calls
  • Server testing with almost no extra efforts
  • Ability to differentiate between server and client issues
  • I don't know... we'll get to know more once we start using it

What do we need?

  • Working PC (yes Windows will do)
  • Working internet connection (we can always try :P)
  • OWASP ZAP - A network monitoring tool
  • Test device (non root will do)
  • Interest

Setting up...

  1. Download OWASP ZAP from here (https://github.com/zaproxy/zaproxy/wiki/Downloads)
  2. Open OWASP ZAP
  3. Choose "Yes, I want to persist this session"

Local Proxy Server Setup

  1. Open Terminal/Command Prompt
  2. Type "ifconfig" on Linux/Unix/Mac, "ipconfig" on Windows
  3. Note your computer's IP address
  4. Go to Tools > Options > Local Proxy in ZAP
  5. Put in your computer's IP Address in the "Address" field
  6. Put something like "8080" in the port field
  7. Click on "OK"

Adding a man (server *) in the middle...

  1. Connect the test device to the same network as your computer
  2. Go to Settings > Wi-Fi
  3. Long press on the connected Wi-Fi
  4. Click on "Modify network"
  5. Open advanced options
  6. Select "Manual" for proxy settings
  7. Put in the IP and port number you entered in OWASP ZAP

And you're done!

Observe the HTTP calls

With this setup you can observe all the HTTP calls any of the apps on your test device are making.

If you wish to monitor the HTTPS calls too, you'll need to take a few extra steps...

Extra mile...

The reason why we use HTTPS is its security. Your network administrator will know the credentials of your online accounts if everyone used HTTP. Some of the most used apps still do, surprise!

To be able to monitor HTTPS calls, you'll need an SSL certificate to authenticate your proxy server. Don't worry, OWASP ZAP helps you with this too!

  1. Open OWASP ZAP
  2. Go to Tools > Options > Dynamic SSL Certificates
  3. Click on save
  4. Copy the saved certificate file onto the test device
  5. On the phone open Settings > Security > Install Certificate From SD Card
  6. Go to the location you copied the certificate at, select it
  7. Give the certificate a name of your choice, say "ZAP"

Enjoy!
